Ejercicio 59 - Botones Deshabilitados

Personalización de Botones Deshabilitados

La propiedad opacity y cursor permiten crear botones con apariencia de deshabilitados, indicando visualmente al usuario que no están disponibles para interacción.

Comparación de Botones Normales y Deshabilitados

Observa las diferencias entre los botones:

  • Botón Normal: Completamente opaco y con cursor de puntero
  • Botón Deshabilitado: Parcialmente transparente y con cursor de "no permitido"

Cómo Funciona la Deshabilitación Visual

opacity: 1.0
opacity: 0.8
opacity: 0.6
opacity: 0.4
opacity: 0.2

Para crear un botón con apariencia de deshabilitado se utilizan dos propiedades principales:

  • opacity: Controla la transparencia del elemento (1 = completamente opaco, 0 = completamente transparente)
  • cursor: Cambia el cursor al pasar por encima del elemento (not-allowed = indica que la acción no está permitida)

Nota: Esta técnica solo cambia la apariencia visual. Para realmente deshabilitar un botón, debes usar el atributo HTML disabled.

Tipos de Cursores

cursor: pointer
cursor: not-allowed
cursor: wait
cursor: help

El cursor puede cambiar para indicar diferentes estados o acciones posibles.

Código CSS

.button {
    background-color: #4CAF50;
    border: none;
    color: white;
    padding: 15px 32px;
    text-align: center;
    text-decoration: none;
    display: inline-block;
    font-size: 16px;
    margin: 4px 2px;
    cursor: pointer;
}

.disabled {
    opacity: 0.6;
    cursor: not-allowed;
}
